<p>From Playwright team</p><p>&nbsp;</p><p><span style="font-size:21px;"><strong>Introduction</strong></span></p><p>&nbsp;</p><p>Playwright tests are simple: they perform actions and assert the state against expectations.</p><p>&nbsp;</p><p>Playwright automatically waits for actionability checks to pass before performing each action. You don't need to add manual waits or deal with race conditions. Playwright assertions are designed to describe expectations that will eventually be met, eliminating flaky timeouts and racy checks.</p><p>&nbsp;</p><p><span style="font-size:21px;"><strong>First test</strong></span></p><p>&nbsp;</p><p>Take a look at the following example to see how to write a test.</p><pre><code class="js javascript js-code">//tests/example.spec.ts

import { test, expect } from '@playwright/test';

test('has title', async ({ page }) =&gt; {
  await page.goto('https://playwright.dev/');

  // Expect a title "to contain" a substring.
  await expect(page).toHaveTitle(/Playwright/);
});

test('get started link', async ({ page }) =&gt; {
  await page.goto('https://playwright.dev/');

  // Click the get started link.
  await page.getByRole('link', { name: 'Get started' }).click();

  // Expects page to have a heading with the name of Installation.
  await expect(page.getByRole('heading', { name: 'Installation' })).toBeVisible();
});</code></pre><p>&nbsp;</p><blockquote><p>Add // @ts-check at the start of each test file when using JavaScript in VS Code to get automatic type checking.</p></blockquote><p>&nbsp;</p><p><span style="font-size:21px;"><strong>Actions</strong></span></p><p>&nbsp;</p><p><span style="font-size:19px;"><strong>Navigation</strong></span></p><p>&nbsp;</p><p>Most tests start by navigating to a URL. After that, the test interacts with page elements.</p><pre><code class="js javascript js-code">await page.goto('https://playwright.dev/');</code></pre><p>&nbsp;</p><p>Playwright waits for the page to reach the load state before continuing. Learn more about page.goto() options.</p><p>&nbsp;</p><p><span style="font-size:19px;"><strong>Interactions</strong></span></p><p>&nbsp;</p><p>Performing actions starts with locating elements. Playwright uses Locators API for that. Locators represent a way to find element(s) on the page at any moment. Learn more about the different types of locators available.</p><p>&nbsp;</p><p>Playwright waits for the element to be actionable before performing the action, so you don't need to wait for it to become available.</p><pre><code class="js javascript js-code">// Create a locator.
const getStarted = page.getByRole('link', { name: 'Get started' });

// Click it.
await getStarted.click();</code></pre><p>&nbsp;</p><p>In most cases, it'll be written in one line:</p><pre><code class="js javascript js-code">await page.getByRole('link', { name: 'Get started' }).click();</code></pre><p>&nbsp;</p><p><span style="font-size:19px;"><strong>Basic actions</strong></span></p><p>&nbsp;</p><p>Here are the most popular Playwright actions. For the complete list, check the Locator API section.</p><p>&nbsp;</p><figure class="table"><table><thead><tr><th>Action</th><th>Description</th></tr></thead><tbody><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-check">locator.check()</a></td><td>Check the input checkbox</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-click">locator.click()</a></td><td>Click the element</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-uncheck">locator.uncheck()</a></td><td>Uncheck the input checkbox</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-hover">locator.hover()</a></td><td>Hover mouse over the element</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-fill">locator.fill()</a></td><td>Fill the form field, input text</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-focus">locator.focus()</a></td><td>Focus the element</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-press">locator.press()</a></td><td>Press single key</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-set-input-files">locator.setInputFiles()</a></td><td>Pick files to upload</td></tr><tr><td><a href="https://playwright.dev/docs/api/class-locator#locator-select-option">locator.selectOption()</a></td><td>Select option in the drop down</td></tr></tbody></table></figure><p>&nbsp;</p><p><span style="font-size:19px;"><strong>Assertions</strong></span></p><p>&nbsp;</p><p>Playwright includes test assertions in the form of expect function. To make an assertion, call expect(value) and choose a matcher that reflects the expectation.</p><p>&nbsp;</p><p>Playwright includes async matchers that wait until the expected condition is met. Using these matchers makes tests non-flaky and resilient. For example, this code waits until the page gets the title containing "Playwright":</p><pre><code class="js javascript js-code">await expect(page).toHaveTitle(/Playwright/);</code></pre><p>&nbsp;</p><p>Here are the most popular async assertions. also includes generic matchers like toEqual, toContain, toBeTruthy that can be used to assert any conditions. These assertions do not use the await keyword as they perform immediate synchronous checks on already available values.</p><pre><code class="js javascript js-code">expect(success).toBeTruthy();</code></pre><p>&nbsp;</p><p><span style="font-size:19px;"><strong>Test Isolation</strong></span></p><p>&nbsp;</p><p>Playwright Test is based on the concept of test fixtures such as the built in page fixture, which is passed into your test. Pages are isolated between tests due to the Browser Context, which is equivalent to a brand new browser profile. Every test gets a fresh environment, even when multiple tests run in a single browser.</p><pre><code class="js javascript js-code">// tests/example.spec.ts
import { test } from '@playwright/test';

test('example test', async ({ page }) =&gt; {
  // "page" belongs to an isolated BrowserContext, created for this specific test.
});

test('another test', async ({ page }) =&gt; {
  // "page" in this second test is completely isolated from the first test.
});</code></pre><p>&nbsp;</p><p><span style="font-size:19px;"><strong>Using Test Hooks</strong></span></p><p>&nbsp;</p><p>You can use various test hooks such as test.describe to declare a group of tests and test.beforeEach and test.afterEach which are executed before/after each test. Other hooks include the test.beforeAll and test.afterAll which are executed once per worker before/after all tests.</p><pre><code class="js javascript js-code">// tests/example.spec.ts
import { test, expect } from '@playwright/test';

test.describe('navigation', () =&gt; {
  test.beforeEach(async ({ page }) =&gt; {
    // Go to the starting url before each test.
    await page.goto('https://playwright.dev/');
  });

  test('main navigation', async ({ page }) =&gt; {
    // Assertions use the expect API.
    await expect(page).toHaveURL('https://playwright.dev/');
  });
});</code></pre>
