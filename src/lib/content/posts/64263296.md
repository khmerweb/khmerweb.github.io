<p>From Microsoft team</p><p>&nbsp;</p><p><strong>Summary</strong>: in this tutorial, you will learn about the TypeScript unknown type to enforce type checking of an unknown value.</p><p>&nbsp;</p><p><span style="font-size:21px;"><strong>Introduction to the TypeScript unknown type</strong></span></p><p>&nbsp;</p><p>In TypeScript, the unknown type can hold a value that is not known upfront but requires type checking.</p><p>&nbsp;</p><p>To declare a variable of the unknown type, you use the following syntax:</p><pre><code class="typescript">let result: unknown;</code></pre><p>&nbsp;</p><p>Like the any type, you can assign any value to a variable of the unknown type. For example:</p><pre><code class="typescript">let result: unknown;

result = 1;
result = 'hello';
result = false;
result = Symbol();
result = { name: 'John' };
result = [1, 2, 3];</code></pre><p>&nbsp;</p><p>Unlike the any type, TypeScript checks the type before performing operations on it.</p><p>&nbsp;</p><p>For example, you cannot call a method or apply an operator on a unknown value. If you attempt to do so, the TypeScript compiler will issue an error:</p><pre><code class="typescript">let result: unknown;
result = [1,2,3];

const total = result.reduce((a: number, b:number ) =&gt; a + b, 0);
console.log(total);</code></pre><p>&nbsp;</p><p>In this example, the result variable has the type of unknown. We assign an array the result value, but its type is still unknown. Therefore, we cannot call the reduce() method of an array on it.</p><p>&nbsp;</p><p>To call the reduce() method on the result variable, you need to use the type assertion to explicitly tell the TypeScript compiler that the type of the result is array. For example:</p><pre><code class="typescript">let result: unknown;
result = [1, 2, 3];

const total = (result as number[]).reduce((a: number, b: number) =&gt; a + b, 0);
console.log(total); // 6</code></pre><p>&nbsp;</p><p>In this example, we explicitly tell the TypeScript compiler that the type of the result is an array of numbers (result as number[]).</p><p>&nbsp;</p><p>Therefore, we can call the reduce() method on the result array without any issues.</p><p>&nbsp;</p><p><span style="font-size:21px;"><strong>Unknown vs Any       console.log('String:', value.toUpperCase());

      console.log('Number:', value.toFixed(2));
      break;

  }
}</code></pre><p>&nbsp;</p><p>In this example, before accessing a method of the value, we validate its type to ensure that the operation is valid.</p><p>&nbsp;</p><p><span style="font-size:21px;"><strong>Summary</strong></span></p><p>&nbsp;</p><ul><li>The unknown type is like any type but more restrictive.</li><li>Use the unknown type to handle data coming from external sources and requires validation before use.</li></ul>
